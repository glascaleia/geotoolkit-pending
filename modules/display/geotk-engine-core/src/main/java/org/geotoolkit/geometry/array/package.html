Implementations of {@link org.opengis.spatialschema.geometry.geometry.PointArray} interface. A minimal {@code PointArray} interface is defined in ISO 19107. This package extends the minimal interface with services targeted for rendering purposes.

This package provides various classes for storing arrays of two-dimensional {@linkplain org.opengis.spatialschema.geometry.DirectPosition positions}. Each class is a compromise between various constraints like JTS compatibility, memory consumption and speed. The simplest implementation is {@link org.geotoolkit.geometry.array.DefaultArray}, which just wraps a float[] array containing (x,y) coordinates. More sophisticated subclasses can compress data in order to hold big coordinate arrays with less memory.

{@link org.geotoolkit.geometry.array.PointArray2D} has an iterator allowing forward-only iteration through coordinate points. It does not allow random access or backward iteration. This limitation makes it easier to implements compression algorithms. As a general rule, users should avoid random access as much as possible.
